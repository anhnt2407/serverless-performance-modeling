SLSPERF: Serverless Computing Performance Model

In this document, we present an interative interface for our performance model for Serverless Computing Platforms. You can take a look at our Github repository for the source code and other artifacts. Here are a few of the SLSPERF characteristics:

Uses Erlang Loss System Underneath

Our performance model uses erlang loss system model for M/G/m/m queues typical in warm server pool of serverless computing platforms.

Employs CTMC Steady-State Calculation

Our model uses CTMC to analyze the steady-state behaviour of the system.

Interactive Model

Our interactive model helps researchers and serverless computing platform users predict the behaviour of their applications in the long run.
The inputs for our model can be classified into two sections: System Properties and Workload Properties.

System Properties

In our approach, the system properties are the administrative characteristics of the serverless computing platform. These properties usually don't differ based on the workload and are common for all workloads hosted on the platform.

How long does the platform keep the servers after being idle? (10 minutes for AWS)

Workload Properties

Workload properties define the workload being deployed on the platform. Our model will then try and predict various properties specific to this workload on the platform.

What is the response time for warm start requests?
What is the response time for cold start requests? (for AWS: user init time + platform init time)
What is the arrival rate? (e.g., 2 requests per second)
